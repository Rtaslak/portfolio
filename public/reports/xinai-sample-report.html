XINAI-001 WordPress REST API User Enumeration Medium Remediated
CVSS v3.1 5.3 · AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:N/A:N · CWE-200 · OWASP A01:2021

Description: The WordPress REST API endpoint /wp-json/wp/v2/users was reachable without authentication and returned the site's author accounts, including their display names and login slugs.

Reproduction:

GET https://www.jasonofbh.net/wp-json/wp/v2/users

Impact: Disclosure of valid usernames removes the guesswork from credential-stuffing and brute-force attacks against wp-login.php, lowering the cost of a future account-takeover attempt.

Remediation: Restrict the REST users endpoint to authenticated requests (filter rest_endpoints or apply a security plugin), and disable public author archives that leak the same slugs.

XINAI-002 Permissive CORS Configuration Medium Remediated
CVSS v3.1 5.4 · AV:N/AC:L/PR:N/UI:R/S:U/C:L/I:L/A:N · CWE-942 · OWASP A05:2021

Description: The application's API reflected an arbitrary Origin request header back in Access-Control-Allow-Origin while also returning Access-Control-Allow-Credentials: true.

Reproduction:

GET /wp-json/ HTTP/1.1
Host: www.jasonofbh.net
Origin: https://evil.example

# Response reflects the attacker origin:
Access-Control-Allow-Origin: https://evil.example
Access-Control-Allow-Credentials: true

Impact: A malicious origin can issue credentialed cross-origin requests and read authenticated responses, exposing data scoped to a logged-in user's session.

Remediation: Enforce a server-side allowlist of trusted origins; never reflect an arbitrary Origin while credentials are allowed. Drop Access-Control-Allow-Credentials wherever it is not strictly required.

XINAI-003 Exposed Development Subdomain Low Remediated
CVSS v3.1 5.3 · AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:N/A:N · CWE-200 · OWASP A05:2021

Description: The host dev.jasonofbh.net was publicly reachable and indexable. Xinai discovered it passively through certificate-transparency logs rather than active brute-forcing.

Reproduction:

# Discover via certificate transparency
curl -s 'https://crt.sh/?q=%25.jasonofbh.net&output=json'
# dev.jasonofbh.net appears in issued certificates; browse to it
curl -I https://dev.jasonofbh.net/

Impact: Pre-production environments frequently run unhardened code, verbose error pages, or stale configuration, exposing functionality and information not intended for the public.

Remediation: Place the development environment behind IP allowlisting or HTTP authentication, add noindex, and exclude it from public DNS and certificate transparency where feasible.

XINAI-004 XML-RPC Interface Enabled Low Remediated
CVSS v3.1 5.3 · AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:L · CWE-799 · OWASP A05:2021

Description: The legacy xmlrpc.php interface was enabled and exposed system.multicall, which lets a client batch many calls into a single request.

Reproduction:

POST /xmlrpc.php HTTP/1.1
Host: www.jasonofbh.net
Content-Type: text/xml

<methodCall><methodName>system.listMethods</methodName></methodCall>
# Response lists system.multicall, wp.getUsersBlogs, pingback.ping ...

Impact: system.multicall amplifies brute-force attempts (hundreds of password guesses per request) and the pingback methods can be abused as a denial-of-service / SSRF reflection vector.

Remediation: Disable XML-RPC entirely if it is unused, or filter the available methods and apply rate limiting at the edge (WAF / reverse proxy).

XINAI-005 Web Technology & Version Disclosure Info Remediated
CVSS v3.1 0.0 (informational) · AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:N/A:N · CWE-200 · OWASP A06:2021

Description: Response headers and the /readme.html file disclosed the exact WordPress core version along with several plugin versions.

Reproduction:

curl -sI https://www.jasonofbh.net/ | grep -iE 'server|x-powered-by'
curl -s  https://www.jasonofbh.net/readme.html | grep -i version

Impact: Precise version fingerprints let an attacker skip discovery and select known exploits matched to the running software, raising the efficiency of a targeted attack. Informational on its own.

Remediation: Suppress the Server and X-Powered-By version strings, remove readme.html, strip the WordPress generator meta tag, and serve generic error pages.

07Recommendations

Prioritized hardening actions, ordered by risk reduction. All listed items were applied to www.jasonofbh.net as part of remediation.

Priority 1 — Reduce the unauthenticated attack surface

  • Lock down the REST API and XML-RPC: require authentication on the REST users endpoint and disable xmlrpc.php (or filter its methods and rate-limit it).
  • Fix the CORS policy: replace origin reflection with a strict server-side allowlist and never combine credentialed responses with arbitrary origins.

Priority 2 — Gate non-production and reduce disclosure

  • Gate the dev subdomain: put dev.jasonofbh.net behind IP allowlisting or HTTP auth and mark it noindex.
  • Suppress version disclosure: remove version headers, the generator meta tag, and readme.html; serve generic error pages.

Priority 3 — Defense in depth

  • Add WAF rules and rate limiting: deploy edge WAF rules for WordPress (login throttling, XML-RPC and REST abuse signatures) and apply per-IP rate limiting to authentication endpoints.
